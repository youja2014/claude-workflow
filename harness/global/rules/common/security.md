# Security Rules

## 시크릿 관리
- API 키, 토큰, 비밀번호를 코드에 절대 하드코딩 금지
- 환경변수 또는 .env 파일로 관리
- .env 파일은 반드시 .gitignore에 포함
- .env.example 파일로 필요한 변수 목록만 문서화

## 입력 검증
- 외부 입력(API 요청, 사용자 입력, 파일)은 항상 검증
- SQL 쿼리: 파라미터 바인딩 사용, 문자열 포매팅 금지
- 파일 경로: path traversal 방지 (../ 차단)

## 의존성
- 알려진 취약점이 있는 버전 사용 금지
- 의존성은 최소한으로 유지
- 사용하지 않는 의존성은 즉시 제거

## 로깅
- 시크릿, 개인정보를 로그에 출력 금지
- 에러 로그에 스택 트레이스 포함
- 프로덕션에서 DEBUG 레벨 비활성화
