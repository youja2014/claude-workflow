### 발생한 이슈 + 해결 (별도 commit 으로 분리)

Phase 1 진행 중 매 Write 호출마다 약 10분씩 hang. 9 파일 = 90분 + 마지막 1 파일 timeout. 사용자가 인지 + 끼어들어 진단 진행.

**근본 원인**: `format-on-save.sh` 의 .md/.ts/.json/... 분기에서 package.json 을 찾기 위해 dirname 으로 위로 올라가는 while 루프. Windows bash 에서 `dirname "D:"` 가 `"D:"` 를 그대로 리턴 → fixed-point 무한 루프 → 외부 timeout (약 10분) 으로 강제 종료. Python 분기는 영향 없음 (`.py` 는 ruff format 만, package.json 탐색 없음).

**패치** (commit `fix(hooks): guard format-on-save against Windows dirname fixed-point` 예정):
- `harness/global/hooks/format-on-save.sh` — `prev_dir` 추적 + while 조건에 `"$proj_dir" != "$prev_dir"` 추가
- `~/.claude/hooks/format-on-save.sh` — 동일 패치 (사용자 글로벌 동기화)
- 검증: 60+s timeout → 0.379s (패치 직후 측정)

**재발 방지**:
- hook 추가 시 hooks 자체를 standalone 으로 실행 가능한 stdin JSON 테스트 케이스 마련 ([[user-CLAUDE.md]] "hook 스크립트 작성" 룰)
- Windows bash 의 `dirname` 동작이 POSIX 와 다른 경계(`D:`, `C:`) 인지 인지, 모든 path-walk 루프는 fixed-point guard 의무화
- Phase 4 (`make eol-check`) 와 별도로, **`make verify` 에 hooks 의 self-test 도 추가 검토** (post-Phase 5 후속)
