# Plans

메타 프로젝트의 작업 분할과 실행 계획.

## 파일

- `wbs.md` — Work Breakdown Structure. 출근 시 우선순위 결정의 원천
- `exec-plans/<name>.md` — 개별 처방·기능의 실행 계획. plan 모드 산출물

## 룰

- `wbs.md` 의 항목은 가능한 한 1-2 세션 단위로 잘게 쪼개기
- 새 처방·기능 시작 시 `exec-plans/<name>.md` 작성 (목표 / 단계 / DoD)
- 완료된 exec-plan 은 삭제하지 말고 그대로 보관 (이력 추적, ADR 보완)
- 완료 시 `status.md` 와 `handoffs/<날짜>.md` 에 결과 반영
