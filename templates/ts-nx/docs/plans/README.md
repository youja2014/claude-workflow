# Plans

프로젝트 목표와 작업 분할.

## 파일

- `goals.md` — 장기 목표 (거의 안 바뀜; 분기/년 단위)
- `wbs.md` — Work Breakdown Structure. 출근 시 우선순위 결정의 원천
- `exec-plans/<feature>.md` — 개별 기능의 실행 계획. plan 모드 산출물

## 룰

- `wbs.md` 의 항목은 가능한 한 1-2일 단위로 잘게 쪼개기
- 새 기능 시작 시 `exec-plans/<feature>.md` 작성 (목표 / 단계 / 의존 / DoD)
- 완료된 exec-plan 은 삭제하지 말고 그대로 보관 (이력 추적)
